### Post 1 — Technical: EKYC Architecture
```
🚀 The EKYC flow I built handles 2M+ users. Here's how I structured it.

When building an EKYC package for a major fintech client, the biggest challenge wasn't the UI — it was making it work reliably across thousands of different Android devices and network conditions.

Here's the architecture that saved us:

1. 🔄 Modular state machine → each step (NID scan, liveness, data validation) is its own state
2. 🛡️ Offline-first → cache progress locally, resume after network failure
3. ⚡ Progressive loading → show skeleton UI while waiting for API
4. 📱 Device-first validation → catch issues before hitting backend

Result: <1% failure rate across 2M+ onboarding sessions.
```
